## fix
NAT readLoop memory bloat — 1 GB resident, 60% CPU on idle browsing
Heap profile from 1.28.16 showed 1052 MB (96% of all live memory) held in NATTable.readLoop's per-entry buffers. Each NAT entry spawned a goroutine with `make([]byte, 65535)`, and entries lived for 60s — so a browser doing hundreds of DNS lookups while watching YouTube ended up with thousands of live entries × 64 KB. The matching alloc_space profile showed 62 GB cumulative through the same path, which is why the GC was running so hard. Three changes: (1) shrink the read buffer from 65 535 to 2 048 bytes — covers any realistic MTU + headroom, drops per-entry footprint by 32×, (2) recycle the buffers via sync.Pool so dial bursts don't allocate, (3) reduce default NAT timeout from 60 s to 10 s for non-voice flows so DNS-style one-shot sockets release quickly. Voice flows still get 120 s. Cleanup interval also tightened from 10 s to 5 s.
