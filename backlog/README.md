# Backlog

Ideas, plans, and deferred tasks that aren't being worked on right now but we want to remember.

One file per topic. File name is the short slug of the idea. Inside each file:

- **What** — short description of the thing
- **Why** — motivation / what problem it solves
- **How** — sketch of the approach (bullet points, pseudocode, files to touch)
- **Cost** — rough effort estimate
- **Blockers** — anything that needs to land first

When an item gets picked up for real work, move it out of here into a task list / PR description and delete the file (or leave a one-liner pointing to the commit).
